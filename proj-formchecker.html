Web Form Monitoring Tool for Client Lead Gen Page

Situation
While working with a client on a new large-scale website, they mentioned an ongoing issue with one of their older landing pages. This page included a ZIP code finder and lead gen form and was the target of their paid ad campaigns. Strangely, the page would randomly stop working - but the rest of the site stayed up.

Several of us tried to troubleshoot it, but it was built on outdated architecture and none of our workarounds (like duplicating the page) fixed the issue. The back-end team was busy building the new site, so our only short-term fix was to manually restart the server when it happened. This meant they were losing ad dollars and leads whenever the form went down without warning.

Task
Even though my main role in the project was mostly project management - keeping the client updated, gathering requirements, supporting developers, and QA testing - I wanted to find a lightweight solution to help protect their marketing ROI and keep leads flowing during the rebuild.

Action
I built a simple, serverless monitoring tool using Azure Functions and SendGrid. It runs once every hour to check if the ZIP code form is still active. If not, it automatically sends an email alert to our team so we can quickly restart the server.

Process
1. Created an Azure Function in JavaScript (Node.js) to check if the page contained the correct form field. Click here to view the code on GitHub.

2. Set up a timer trigger (0 0 * * * *) to run the function once per hour.

3. Connected a low-cost Azure Blob Storage account to support execution and logging.

4. Created a free SendGrid account, verified my domain, and added the API key to Azure securely.

5. Implemented native HTTPS logic to call SendGrid directly (instead of importing external libraries) for portal compatibility.

6. Tested the setup using fake field names to confirm email alerts worked as expected.

Challenges & Lessons Learned
Storage Dependency - Even though Azure Functions are “serverless,” they still need Blob Storage to manage timer state and logs.

Module Limits - I initially tried to use @sendgrid/mail, but learned that external packages can’t be installed directly in the portal. I rewrote the code using native modules like https instead.

Results
This solution worked extremely well. It cost almost nothing to run, sent real-time alerts quickly after the form went down, and let our team respond within minutes - almost always before the client even noticed.

It saved the client thousands of dollars in wasted ad spend and prevented lead loss during a critical marketing window. It also allowed our back-end development team to stay focused on building their new site without dropping everything every time the old page broke.

The client was thrilled, extended their support agreement post-launch, and is now planning another large-scale project with us. This quick fix turned into a long-term win - for both their business and our team’s relationship.
